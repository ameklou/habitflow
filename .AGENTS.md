# AGENTS.md

## Project Context

This repository is a Ruby on Rails + Hotwire application for a habit tracker named HabitFlow.

HabitFlow helps users create habits, track daily progress, maintain streaks, review statistics, receive reminders, and build consistency over time.

Codex must treat this app as a production-grade Rails application, not a prototype.

## Role

## Core Rules

- Prefer Rails conventions over custom architecture.
- Prefer Hotwire over heavy JavaScript.
- Keep controllers thin.
- Keep views simple and accessible.
- Put complex business workflows in service objects.
- Use authorization for all user-owned resources.
- Do not add gems without approval.
- Do not rewrite large parts of the app unless explicitly requested.
- Do not make destructive database changes without approval.
- Always protect against IDOR vulnerabilities.

## Rails Standards

Use RESTful controllers where possible:

- `index`
- `show`
- `new`
- `create`
- `edit`
- `update`
- `destroy`

Prefer scoped queries:

```ruby
current_user.habits.find(params[:id])
```

Avoid unsafe global lookups:

```ruby
Habit.find(params[:id])
```

## Hotwire Standards

Use:

- Turbo Drive for navigation
- Turbo Frames for partial page updates
- Turbo Streams for real-time or multi-target updates
- Stimulus for small client-side behavior

Avoid SPA architecture.

Stimulus controllers must be small and focused.

## Security Standards

Always consider:

- Authentication
- Authorization
- Strong parameters
- Input validation
- SQL injection
- XSS
- IDOR
- Sensitive data exposure
- Secure session handling

Never log secrets or tokens.

## Testing Standards

Use RSpec unless the project already uses another test framework.

Expected commands:

```bash
bundle exec rspec
bundle exec rubocop
bundle exec brakeman
```

Run focused tests first.

Do not claim tests passed unless they were actually run.
